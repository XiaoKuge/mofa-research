# ChatGPT's 大语言模型Deep Research功能综合分析报告

# Executive Summary

We compare the advanced **Deep Research** capabilities of major AI platforms – Google Gemini, OpenAI ChatGPT, Anthropic Claude, Perplexity AI, Moonshot AI’s Kimi, and China’s DeepSeek – along multiple dimensions. All of these systems augment large language models (LLMs) with external search and retrieval, but they differ in data freshness, tools, context size, explainability, and enterprise features. For example, Google’s Gemini Deep Research agent autonomously plans and executes multi-step searches over the web and private data (Gmail, Drive, Chat), producing multi-page, cited reports【8†L183-L189】【14†L49-L57】. ChatGPT has introduced an integrated **Search** mode (GPT-4o) that fetches up-to-date web answers with live source links【25†L51-L59】. Anthropic’s Claude offers a “Research” mode that performs iterative searches and returns answers with “easy-to-check citations”【27†L41-L48】. Perplexity functions as a real-time AI search engine, synthesizing web content into direct answers with footnoted sources【34†L203-L210】. Moonshot’s Kimi provides extremely long-context reasoning (up to 256K tokens) and launched an AI-powered **Explore** mode (Oct 2024) that “deeply reads” hundreds of web pages per query【19†L104-L112】. DeepSeek (China) releases open MoE models (V3, R1, V3.1, etc.) with 128K token windows and strong reasoning modes【40†L315-L324】, though its research use is primarily via API. 

In short, **Gemini** shines in enterprise/Workspace-integrated research with Google search and user data (long context, multimodal, chain-of-thought via a “thinking” panel)【14†L69-L76】【8†L183-L189】. **ChatGPT (OpenAI)** offers broad web search with source links and a huge plugin ecosystem, making it ideal for general queries and multi-turn research【25†L51-L59】【25†L99-L107】. **Claude** focuses on safe, iterative exploration (including “extended thinking” and web tools), suited for complex research tasks in enterprise settings【27†L41-L48】【29†L216-L224】. **Perplexity** is built specifically for research: it uses continuously-updated web indexes and provides concise, cited answers and connectors (e.g. Google Drive)【34†L203-L210】【36†L49-L58】. **Kimi** targets extreme-long-context scenarios and research in the Chinese market: its “Explore” mode aggressively crawls hundreds of pages, and its models boast trillion-parameter MoEs with 256K context【19†L104-L112】【16†L204-L207】. **DeepSeek** offers open LLMs with large context and dedicated reasoning modes (R1 with chain-of-thought), aimed at developers building agentic workflows【40†L315-L324】【22†L45-L53】. 

Below we analyze each dimension in detail, with references. Our final comparison table highlights each vendor’s strengths, weaknesses, and ideal use-cases. All information is drawn from official docs, technical blogs, and reputable sources (citations in brackets). 

## Data Sources & Freshness

- **Google Gemini (Deep Research)** draws on **real-time web search** (Google Search) plus optionally the user’s private data (Gmail, Drive, Chat). It “autonomously searches and deeply browses the web … to find relevant, up-to-date information”【14†L62-L70】. Gemini’s model is periodically updated (Gemini 3.x series in 2025–2026) and can include very fresh news or facts, though it uses the standard Google Search engine. 
- **ChatGPT (Search mode)** leverages third-party web search providers (e.g. Bing) and premium news/content partnerships. The ChatGPT Search model (GPT‑4o) was “post-trained using … synthetic data” and explicitly “leverages third-party search providers, as well as content provided directly by our partners, to provide the information users are looking for”【25†L139-L144】. This yields answers based on current internet content (scores of live websites, stock quotes, weather, etc.). 
- **Claude (Research)** similarly requires enabling web search and optionally workspace data (Google Workspace, etc.). With Research on, “Claude will kick off the research process across your internal context (Gmail, Calendar, Docs) and the web,” conducting iterative searches【27†L59-L64】. Anthropic’s API also offers a **Web Search tool** giving “direct access to real-time web content, allowing it to answer questions with up-to-date information beyond its knowledge cutoff”【29†L216-L224】. 
- **Perplexity AI** maintains its own continuously-refreshed web index and crawlers. By design, it is a search engine, “incorporat[ing] real-time web search capabilities” so that responses are always based on **current Internet content**, citing sources【34†L203-L210】. (Note: Perplexity’s scraping practices have drawn legal scrutiny【34†L213-L220】.) Perplexity can also connect to personal data via “App Connectors” (e.g. Google Drive) to search user files【36†L47-L56】. 
- **Kimi (Moonshot AI)** offers an “Explore Edition” search mode since Oct 2024. Kimi Explore “adopted a different approach: letting the AI search autonomously … reading through hundreds of pages”【19†L104-L112】. In effect, one query dispatches thousands of parallel searches (Kimi claims 10× the queries of normal search) and reads up to ~500 pages per query【19†L104-L112】【19†L124-L130】. It likely uses Bing-like search or an internal crawler but the details are proprietary. Kimi also now has **Kimi-Researcher** (mid-2025), an autonomous agentic model that performs multi-turn search (around 23 steps, 200+ URLs) via an internal parallel search tool【22†L10-L18】【22†L45-L53】. 
- **DeepSeek (China)** provides powerful LLMs but no public information suggests a dedicated multi-search agent UI. Its models (V3 series, R1) are trained on vast corpora and support tool use, but their web-search integration is likely developer-driven (via RAG or API). DeepSeek models can work with retrieval systems but details are scarce. The BentoML guide notes DeepSeek’s R1 model is specialized for “complex math, coding, and reasoning” with chain-of-thought【40†L315-L324】. DeepSeek has released vision-language models (DeepSeek-VL) but not specifically a search agent.

## Retrieval Methods

- **Retrieval-Augmented Generation (RAG):** All these systems use RAG or agentic retrieval to incorporate external information. ChatGPT’s Search and Claude’s Research both run real-time search queries in the loop, effectively RAG with web sources. Gemini’s Deep Research agent “plans -> search -> read -> iterate”【12†L849-L857】. Perplexity’s engine runs search queries and feeds results to the LLM for summarization with citations. DeepSeek and Kimi models support tool use (with code execution or browsing tools) that enable RAG. 
- **Vector/Embeddings:** Claude’s API does not publicly document a proprietary vector search, but it does allow a code tool for filtering search results. Perplexity’s products may use vector retrieval on their own updated indexes for fast search, but this is internal. DeepSeek-V3 and R1 are MoE LLMs; developers can use vector search over embeddings of documents as needed. 
- **Web Crawling:** Perplexity explicitly runs web crawlers (albeit controversially【34†L213-L220】). Gemini relies on Google’s search index, which constantly crawls the web. Claude’s web search likely uses Bing or a similar API. Kimi Explore likely issues queries to existing search APIs at massive scale. DeepSeek does not publish crawlers. 
- **Proprietary Corpora:** All base models are pre-trained on large text corpora (Common Crawl, etc.), but the freshness in answers comes from live search. Claude and ChatGPT also have fine-tuning on curated data, but do not train on user queries. Gemini as a product is continuously updated by Google’s training data and web knowledge graph. DeepSeek’s models are open (e.g. DeepSeek-V3-0324) with public checkpoints and have been updated via reinforcement learning【40†L278-L287】.

## Citation & Evidence Tracing

- **Gemini**: In Deep Research mode, Gemini returns **detailed reports with citations**. The API documentation states it produces “reports … with detailed, cited sources”【8†L183-L189】. The Google UI (“Deep Research” in the Gemini app) includes a *thinking panel* and, presumably, an output that cites URLs or blocks of text. (Gemini is internally built on Google Search, so results can be linked.) 
- **ChatGPT**: The new ChatGPT Search explicitly provides **links to sources**. After answering a query, ChatGPT shows clickable source links (news articles, websites) at the top of the response【25†L100-L107】. Users can open a “Sources” sidebar for full references【25†L100-L107】. (The images below illustrate this: ChatGPT answered a weather question with a linked AccuWeather source【25†L100-L107】 and shows a “Sources” panel.) This makes ChatGPT answers verifiable. 
- **Claude**: Claude’s Research mode also returns citations. The Help Center notes Claude delivers “thorough answers … complete with easy-to-check citations”【27†L46-L48】. In its API web-search tool, Claude “automatically cites sources from search results as part of its answer”【29†L216-L224】. Thus Claude’s responses include footnotes or URL references to the websites it used. 
- **Perplexity**: Citing sources is a core feature. Perplexity’s answers are typically presented with numbered footnotes linking to each source【34†L203-L210】. For example, an answer might say “… based on [Source Title][1]” with a link. (This Wikipedia summary confirms Perplexity “synthesizes responses … citing sources used”【34†L203-L210】.) The connectivity to Google Drive files also allows citing documents. 
- **Kimi**: The documentation does not clearly state a citation format. The Chinese news on Kimi Explore highlights its high-volume search but does not mention citing. It seems Kimi’s answers are delivered directly (like standard chatbots) without formal source links. The focus is on completeness and multi-page output rather than traceability. (The Kimi-Researcher paper does not detail citation style, focusing on learning via reinforcement learning【22†L10-L18】.) 
- **DeepSeek**: DeepSeek’s products are model-centric (V3, R1, etc.). They do not natively produce linked citations unless developers implement their own retrieval pipelines. Their inferences are “direct answers” or chain-of-thought text, without explicit source links. (The BentoML guide distinguishes DeepSeek-V3’s “Direct answers” vs R1’s step-by-step reasoning【40†L315-L324】, but no mention of citing.) 

## Context Window & Long-Document Handling

- **Google Gemini** boasts an enormous context window. As of Gemini 3.1 Pro, **1 million tokens** are supported in beta【7†L214-L222】. (Gemini 3.0 Ultra already had 256K tokens base.) This far exceeds other publicly-available models, enabling Gemini to ingest and reason over very long documents or even entire books. 
- **Claude** has also extended context. Claude Opus 4.6 (Feb 2026) supports up to **1M tokens** (beta)【31†L22-L26】, with 200K normally. (Claude Sonnet/Opus series have sub-variants like 4.5 with 128K, and newer 4.6 with 1M.) This allows lengthy chats, documents or codebases. Claude also offers “compaction” to summarize older context【31†L63-L70】. 
- **OpenAI ChatGPT** (GPT-4 Turbo/4o) supports up to about **128K tokens** (as of late 2025)【49†L7-L11】. This is very large compared to GPT-3-era models. (Earlier GPT-4 Turbo was 32K, now extended to 128K.) ChatGPT’s Search mode (GPT-4o) can handle long chats and combine multiple answers, but still under this window. 
- **Kimi** was the first with **128K tokens** (Nov 2023) and has since moved to **256K tokens** (Sep 2025)【16†L204-L207】. Its “context caching” and streaming allow effectively unlimited context by chunking. Kimi-K2 (1T param MoE) and later K2.5 support 256K tokens, making it unique for extremely long research sessions. Kimi’s “OK Computer” mode can also ingest millions of data points concurrently【16†L209-L217】. 
- **Perplexity**: No official context window is published, but it uses frontier LLMs under the hood (e.g. GPT-5.2 for long contexts as cited by Perplexity Computer)【37†L153-L160】. Likely comparable to ChatGPT on its highest tier. For per-query research, Perplexity splits content across browsed pages and synthesizes, so it effectively handles large content by multi-query aggregation. 
- **DeepSeek** (Moonshot’s models) supports up to **128K tokens** (DeepSeek-V3, R1, V3.1)【40†L315-L324】. The table shows all DeepSeek variants have 128K windows. (These open models emphasize “thoughtful responses” rather than memory length.) Without an API like Gemini’s agent, users must manually chunk text. 

## Multimodal Support

- **Gemini** (Google) is natively multimodal. Gemini models can process text, images, audio, and video【7†L242-L245】. The Deep Research agent can take images or mixed media inputs (though this may increase cost). Google also offers Canvas (interactive reports) and image/video gen in the Gemini app. 
- **Claude** is multimodal in its latest versions. Sonnet 4 models handle images and text, and Opus can integrate with tools like Claude in Excel (data integration). Claude’s vision capabilities are more limited compared to Gemini (no direct user UI for images yet, aside from Claude-in-Excel). 
- **ChatGPT** supports multimodal interactions in certain interfaces. The web ChatGPT has an **Image** tool (DALL·E) and a code interpreter (files, images, charts) for Plus and Enterprise users. Its conversation can include image uploads and audio (Advanced Voice)【25†L146-L154】. ChatGPT Search also has specialized answers for weather, stocks, etc., with visual panels【25†L87-L96】. 
- **Perplexity** is primarily text-based. It does have an experimental “video generation” feature (as mentioned in Pro plan), but its core Q&A is text-only. It cannot natively ingest images or other modalities in search results. 
- **Kimi** explicitly has vision. Kimi K2 models are **multimodal** (image + text)【3†L220-L223】. The company is testing AI video generation【15†L181-L188】 and has an image understanding feature (Kimi-VL). In practice, the public Kimi chatbot can take image inputs and generate image/video outputs in development. 
- **DeepSeek** has released vision-language models (DeepSeek-VL, DeepSeek-R1-0528 supports vision). However, DeepSeek’s primary offerings are language-only. Their documentation highlights text tasks; any image/video work would require a separate model (not core to “Deep Research” function). 

## Reasoning & Chain-of-Thought Transparency

- **Gemini** explicitly exposes its reasoning steps in Deep Research. Google introduced a “thinking” panel so users see what Gemini learned so far and its next moves【14†L153-L156】. The Deep Research pipeline itself involves iterative self-critique and multiple passes【14†L158-L162】, though the final report is a polished synthesis. 
- **ChatGPT** does not show its internal chain-of-thought to users. It uses GPT‑4’s reasoning capabilities internally, but the answer is delivered as a final narrative (no step-by-step log). However, one can prompt ChatGPT for step-by-step answers manually. The “Search” mode does not reveal internal steps, only the final answer with sources. 
- **Claude** similarly does not publicly reveal its chains of thought. Its “extended thinking” tools can let it run longer reasoning (e.g. “parrot mode”), but users see only the final answer. Claude emphasizes safe reasoning rather than transparency. (In API, one can ask for more explanation, but it’s not a built-in UI feature.) 
- **Perplexity** prioritizes concise answers. It generally does *not* provide step-by-step reasoning, only a brief summary and citations. Its “Answer Engine” is akin to search results, not a thought-process log. 
- **Kimi** does have a form of transparency. The Kimi-Researcher model, for example, defines a (think_t, action_t) output at each step【22†L45-L53】, and Kimi’s UI (like Gemini) shows a “thinking mode” where it can output intermediate reasoning if prompted. In their app, responses can include a “|COT mode” (chain-of-thought) option. 
- **DeepSeek** places a special emphasis on internal reasoning. Its R1 model is designed for chain-of-thought and can be prompted to show its reasoning steps (using `<think>` tokens or system instructions)【40†L315-L324】. DeepSeek-V3.1 even has a hybrid mode where you can switch between “thinking” and “non-thinking” behavior【40†L284-L293】. Thus DeepSeek is explicitly aimed at making reasoning explicit when desired.

## Customization, Plugins & API Access

- **ChatGPT** (OpenAI) has a rich ecosystem. Users can install **plugins** (web browser, code, third-party apps) and developers can integrate via the GPT-4/4o API. ChatGPT offers a browser extension and supports tools like Code Interpreter, Python REPL, etc. Its API and plugins allow deep customization (Enterprise accounts have build-your-own GPTs). 
- **Gemini** is primarily accessed through Google’s ecosystem. For end users, Deep Research is built into the Gemini app (free for consumers, additional access for Google Workspace). For developers, Gemini models (3.1 Pro etc.) are available via the Google Cloud Generative AI API (GenAI)【12†L860-L869】. The Deep Research agent itself is accessed via the **Interactions API**【12†L860-L868】 and currently cannot take custom tools from users. 
- **Claude** provides an API (Anthropic Claude API) with tool use. Developers can use **Web Search** and **Code** tools via API (as documented【29†L216-L224】【29†L253-L261】). Anthropic also offers **Claude In Excel/PowerPoint** and a Connectors catalog (e.g. Slack, Salesforce) for enterprises. The Claude Chat UI supports a “Research” toggle (on Pro/Team plans) but not end-user plugins. 
- **Perplexity** has an **API** (Search API and Assistant API) that returns JSON with answers and source URLs. It also offers “Perplexity Connectors” (e.g. Google Drive) so that organizations can query their own documents【36†L47-L56】. The architecture is closed (no user plugins), but developers can script queries. Perplexity Max (new) even allows orchestrating multiple models (ChatGPT, Claude, Gemini) behind the scenes【37†L151-L160】. 
- **Kimi (Moonshot AI)**: The public Kimi app/website is the main interface (web, iOS, Android). Moonshot also offers an API platform (platform.moonshot.ai) where developers can call their models. Kimi-VL and K2 are open-sourced under MIT-like licenses【15†L149-L158】【16†L200-L208】, so teams can run DeepSeek-compatible inference. Advanced agentic features (like “OK Computer” for slides/websites) exist in the app. The Kimi-Researcher is accessible via Kimi’s UI. 
- **DeepSeek**: DeepSeek provides an API for its models (via deepseek.com) and open-source releases (DeepSeek-V3, R1, V3.1 on Hugging Face【40†L307-L315】). There is no end-user plugin system – it’s a toolkit for developers. Because the models (especially R1/V3.1) are available, teams can build custom agents around them. BentoML’s guide shows how to integrate DeepSeek models with code for “agentic workflows”【39†L75-L83】.

## User Experience & Query Types

- **ChatGPT**: A chat interface supporting free-form queries. With Search enabled, users can click a globe icon or type “/search” to run web queries. Answers are conversational, and a “Sources” panel can open showing references【25†L100-L107】. Plus and Enterprise users get immediate access to Search; free users get it later【25†L39-L47】. ChatGPT also supports multi-turn follow-ups and diverse output formats (text, code, tables, images). 
- **Google Gemini**: In Gemini’s web/mobile app, users select “Deep Research” from the prompt bar. They enter a topic (e.g. “market analysis of X”), and Gemini autonomously breaks it down【14†L57-L66】. The UI shows the plan and a “thinking” panel as the agent works through sources【14†L153-L156】. The final product is a multi-page report (with possibly an Audio Overview) that the user can navigate. (Gemini is integrated into Google Workspace, so eligible users see it seamlessly.) 
- **Claude**: On claude.ai or desktop apps, a user toggles the “Research” button (bottom left)【27†L51-L59】. Claude then automatically performs the multi-search process. Users can also prompt Claude in regular mode to “use the Research tool” if needed. Output is a single comprehensive answer (often long-form paragraphs) with citations. There is no special step-by-step display, but users can refine the prompt or ask for clarifications. 
- **Perplexity**: Operates like a search engine. The user types a question and gets an answer box with sources at the bottom. It also shows a pane listing the key sources used (clickable). Perplexity Pro/Max adds features like setting the agent’s “persona” and connecting to private data. The interface is web-based (with mobile apps and browser extensions). Users can also upload documents or connect apps; the query can then cover web+private data jointly【36†L49-L58】. 
- **Kimi**: The Kimi chatbot (kimi.com) is primarily Chinese-language but supports English. The “Explore” mode is akin to Perplexity: a search box where AI fetches and summarizes many pages. Results are usually lengthy summaries or reports. Kimi also has an “OK Computer” mode to generate webpages/slides, and a “Slides” tool. The interface allows switching between human-like chat and agentic modes (instant vs thinking). Advanced Chinese users can prompt for chain-of-thought or use Kimi-Researcher for deep tasks. 
- **DeepSeek**: Has no consumer UI. Users (typically developers) submit prompts via API or a Python client. There are examples in documentation for “scientific reasoning” or “math problems.” Query types include Q&A, summarization, code generation, etc. Without a built-in search, DeepSeek users must explicitly provide context (documents, or use a separate retrieval step) for evidence-based answers. 

## Accuracy, Hallucinations & Factuality

- **ChatGPT**: With its Search mode, factuality is tied to source quality. It generally **provides source links** so answers can be checked, reducing hallucination【25†L51-L59】【25†L100-L107】. Nonetheless, ChatGPT can still make mistakes if it misinterprets sources or the prompt. OpenAI continuously tunes GPT‑4 with synthetic data to improve accuracy. 
- **Claude**: Anthropic emphasizes safety and factuality. The Claude 4.6 announcement highlights “overall safety profile as good as, or better than, any other frontier model”【31†L43-L47】. The web search tool includes dynamic filtering and code execution to improve accuracy by weeding out irrelevant content【29†L270-L279】. In practice, Claude Research answers tend to be thorough and aligned, though Claude can also err if sources are ambiguous. 
- **Perplexity**: Strives for precision. Its answers are concise and *sourced*, which users find helpful for verification. However, some media have noted legal concerns about content scraping【34†L213-L220】, and like any AI, Perplexity can “hallucinate” if it mis-synthesizes sources. Perplexity Pro’s feature of “10× citation density”【48†L270-L273】 suggests extra emphasis on sourcing. 
- **Google Gemini**: Google has powerful retrieval, but does not claim zero hallucination. The Deep Research agent is in preview, and Google cautions that it should be fact-checked (“review the citations provided”【12†L895-L903】). However, Gemini’s backbone on Google Search and Knowledge Graph should help accuracy. Early reports suggest Gemini Deep Research produces high-quality comparative analysis (a new category of output), but concrete metrics are not public. 
- **Kimi**: The Moonshot team claims strong performance (e.g. matching GPT-4 in math/coding【15†L183-L188】). Chinese news reports also claim they have “basically eliminated” hallucination【16†L247-L254】 (though that’s unverified PR). Kimi’s Explore, by scanning 500 pages per query, may reduce omission errors but could still summarize incorrectly. There’s no independent study of Kimi’s factuality in English. 
- **DeepSeek**: The BentoML article notes DeepSeek-R1’s high reasoning but also explicitly says hallucination was reduced by ~45–50% in R1-0528 compared to earlier models【40†L257-L264】. DeepSeek is open-source, so accuracy depends on model choice: R1 and V3.1-Think are optimized for correctness at the cost of verbosity. Without a built-in retrieval, any hallucination likely comes from the base LLM rather than external mis-citation. 

## Privacy, Data Retention & Enterprise Controls

- **ChatGPT**: OpenAI’s policy is that data is not used to train models (especially in Enterprise/Edu) and users can opt out of sharing. ChatGPT Enterprise offers features like SSO, data logs, and admin control. The Search mode likely logs queries to partners (Vox Media, etc.) as per OpenAI’s publisher agreements【25†L129-L138】, but user data is isolated under OpenAI’s policies. 
- **Gemini**: As a Google product, Gemini Deep Research in Workspace means enterprise admin controls (data lives in Google Cloud). Google states that search and uploaded documents are not used to train models except for safety. The docs warn about security: don’t upload untrusted docs, vet malicious pages【12†L886-L895】. Google enforces privacy and security policies (e.g. you must enable Workspace Data integration explicitly). 
- **Claude**: Anthropic offers **Zero Data Retention (ZDR)** options. The web search tool is “ZDR-eligible” – data passed through it is not stored after the response【29†L228-L230】. Claude in Enterprise also can be set to not train on inputs, and offers SOC 2 compliance. The Help Center notes that research queries count against normal usage limits, but does not detail retention, implying normal Claude policies apply (no training on corporate data by default). 
- **Perplexity**: It markets itself to enterprises with SOC2 Type II, encryption, and an explicit guarantee that “no training is done on customer data”【48†L291-L294】. The Google Drive Connector respects Google permissions【36†L49-L58】 and syncs files privately. Enterprise plans include audit logging and data retention controls【36†L49-L58】【48†L285-L293】. However, Perplexity’s own query logs might still be used to improve its search algorithms (this is not fully public). 
- **Kimi**: Moonshot has not published detailed enterprise policies. As a Chinese company, it likely stores user data. The Kimi app is free with rate limits; premium tiers offer more usage. There are no public statements about data retention or enterprise controls. Given China’s legal environment, Kimi’s data may be subject to Chinese regulations. 
- **DeepSeek**: As an open model/API provider, DeepSeek’s data policies are determined by the company or the user’s own deployment. If using DeepSeek’s API, presumably queries are logged by the service (they haven’t highlighted a ZDR option). If self-hosting the open models, data stays on-prem. In summary, privacy depends on deployment choice. 

## Pricing & Limits

- **ChatGPT**: Offers a free tier and a ChatGPT Plus subscription (~$20/month) for GPT-4 access. ChatGPT Search (GPT-4o) is being rolled out to Plus, Teams, and eventually all users【25†L39-L47】. Enterprise/Edu customers have custom pricing. The backend API (GPT-4o) is billed by tokens (currently ~$5/$25 per million tokens for GPT-4 via OpenAI).
- **Gemini**: In the Google AI API (GenAI) model, pricing is pay-as-you-go on top of Gemini’s cost【12†L860-L869】. Example estimates: a typical Deep Research task (multiple searches and ~60K output tokens) costs a few dollars【12†L874-L881】. The user-facing Gemini app has a free level; Workspace users may get it included or on Workspace licensing. (Google offered free Deep Research access to students.) 
- **Claude**: Claude’s API is priced at $5 per million input tokens and $25 per million output tokens (consistent with [31†L78-L81]). Claude “Pro” (for individuals) or “Max” (for power users) subscriptions exist (e.g. Claude Pro at $20/mo for 100K context model, Max for 200K). Usage limits (rate and total tokens per day) vary by plan; Research mode simply consumes more tokens per answer. 
- **Perplexity**: Free version allows a limited number of daily queries. **Perplexity Pro** is $20/month ($200/yr) and grants “unlimited Pro queries” with multi-step search【48†L253-L262】. **Perplexity Max** (new, Feb 2026) costs $200/month and adds advanced model access and features. Enterprise seats run $40–$325 per user/month (for Pro/Max tiers)【48†L274-L283】【48†L299-L308】, with heavy usage quotas. 
- **Kimi**: The Kimi apps are free with basic rate limits (e.g. 5 free searches per day【19†L114-L122】). Premium subscriptions (“Moderato”, “Allegretto”, “Vivace”) offer higher usage, faster service, and access to Kimi’s advanced models (K2 Turbo, Kimi-Researcher, etc.)【16†L229-L237】. Exact prices are not public (likely tiered around tens of USD monthly). Enterprise options are not clearly defined. 
- **DeepSeek**: DeepSeek’s APIs and models appear to be mostly free or open (MIT license) for the core LLM weights【40†L307-L315】. They likely charge for hosted API usage, but details are scarce. Since DeepSeek is research-oriented, cost may be similar to self-hosting on consumer cloud. In any case, limits would follow usual token billing. 

## Recent Major Updates (18 months)
